**bloqué par le Secure Event Input** d'un panneau keychain d'une autre app pendant
toute la session. Les **captures d'écran** ont confirmé le *rendu*, l'*arbre AX* et
l'*absence de crash* ; l'*interaction* est couverte par les tests ViewModel mais
**pas encore pilotée à l'écran**. À refaire dans une session où l'entrée sécurisée
n'est pas captée (aucun champ mot de passe actif) :

  déplier/enfant), ⏎ (zoom/open), **⌘⌫ (corbeille de la sélection, gaté)**, espace
  (Quick Look), type-select. + **multi-sélection** ⇧/⌘-clic et **corbeille de masse**.
  → clic sur une ligne pour donner le focus à la table, puis `key code` ; vérifier le
  surlignage qui bouge et le total qui décrémente après ⌘⌫.
- **P0 🖥️ SPEC-01 — clic/double-clic** : confirmer que le clic simple sélectionne
  (routage `hitTest` déterministe) et que le double-clic zoome/ouvre. *Non re-confirmé
  après le fix `hitTest`* faute d'événements synthétisés fonctionnels.
- **P1 🖥️ SPEC-03 — bascule Exact/Attribution** : vérifier le re-scan au toggle et
  que le total change du bon montant sur une fixture à hardlinks (le test golden le
  prouve déjà hors-GUI).
- **P1 🖥️ SPEC-03 — panneau Réconciliation sur un vrai volume** : ouvrir le popover
  « ? », vérifier que scan + corbeille + purgeable + non-attribué ≈ « used (API) », et
  comparer à `df`. Les chiffres corbeille/snapshots/purgeable ne sont **pas** testés
  unitairement (touchent le FS réel) — à valider à l'œil.
- **P1 🖥️ SPEC-06 — sheet Remote** : la carte « Connect to a server… » et le
  formulaire n'ont été **vus** que build-vérifiés (sous le pli du splash) ; ouvrir la

- **P2 🖥️ VoiceOver / daltonisme / Larger Text** : passer un vrai lecteur d'écran

## 2. Écarts fonctionnels assumés (documentés dans les specs)

- **⚠️ SPEC-02 — `extStats` de `invalidate` best-effort** : pour une extension
  **partagée entre sous-arbres ET modifiée** dans le sous-arbre re-scanné, le delta
  n'est pas exact (la sous-contribution d'origine n'est pas stockée par nœud — parti
  low-RAM). Tailles/comptes/dirCount restent exacts ; le **chemin suppression (A6)
  est exact**. *Cure possible* : maintenir `extStats` par sous-arbre de 1ᵉʳ niveau
  (Axe C du plan) — surcoût d'état, à trancher.
- **⚠️ SPEC-03 / A4 — clones APFS non déduits** : comptés pleins dans les deux modes
  (note UI honnête). Adopter `ATTR_CMNEXT_PRIVATESIZE` exige un parsing d'attributs
  étendus (`forkattr`) **plus risqué** que le buffer standard. 🔬 **à prototyper +
  mesurer** disponibilité/coût/sémantique avant adoption ; golden test avec `cp -c`.
- **⚠️ SPEC-04 — D1** surfacé par la **bannière** (pas de modal par-suppression) ;
  **pastille dirty** sur les lignes-dossier de la liste uniquement (pas sur les tuiles
  treemap). 🔬 Volume d'events en scannant `/` (débounce 1 s en place) — à observer sur
  un gros arbre actif (caches).
- **⚠️ SPEC-05** — le `fileCache` est **borné et réutilisé**, pas explicitement vidé
  au zoom-out (le budget mémoire reste plafonné). Calibrer **B1 vs B2** (fichiers du
  seul zoomRoot vs +1 niveau) sur de vrais dossiers denses.
- **⚠️ SPEC-06** — auth SSH **par clé uniquement** (BatchMode ; pas de mot de passe ni
  config agent au-delà d'`-i identity`). Backends **Time Machine / archives** non
  construits (le contrat les prévoit).
- **⚠️ SPEC-07** — **panneau About custom non fait** (la version vit dans l'Info.plist ;
  pas d'écran About dédié). Cask Homebrew + auto-update hors v1.
- **⚠️ SPEC-08 / J10.3 — tailles de texte figées** : ~200 `.system(size:)` fixes ;
  rendre l'app réactive à « Larger Text » est un **rétrofit design global** (chantier
  dédié, calibrage anti-casse de la densité). **Non fait.**
- **⚠️ SPEC-08 / J10.4 — traduction** : le *bug* de locale mixte (J9.5) est réglé ; la
  **traduction** dans d'autres langues (String Catalog `.xcstrings` peuplé) reste à
  faire. Le treemap n'est pas **navigable tuile-à-tuile** au rotor VoiceOver (résumé
  parlé seulement).
- **SPEC-01 / J3.7** — en-têtes de **colonnes triables** (nom/taille/nb/date) :
  explicitement **optionnel** ; l'infra `NSTableView` est prête (`sortDescriptors`).

## 3. Prérequis externes (bloquants hors-code)

- **⛔ SPEC-07 — signature Developer ID + notarisation non exécutées** : `release.sh`
  est complet mais exige un **compte Apple Developer** (cert « Developer ID
  Application »), un **profil `notarytool`** et `gh auth`. À faire une fois :
  - `git tag v0.1.0` puis `DEVELOPER_ID="Developer ID Application: …" ./release.sh`
  - vérifier `spctl -a -vv -t open …` → « Notarized Developer ID »
  - **télécharger le DMG sur une AUTRE machine/session** → ouverture sans « développeur
    non identifié ».
  - vérifier que le **grant Full Disk Access persiste** après un rebuild signé (cdhash
    stable) — bénéfice clé vs l'ad-hoc actuel.
- **⛔ SPEC-06 — scan SSH réel** : nécessite un hôte avec **Remote Login** + **GNU
  `find`/`-printf`** (le `find` de macOS ne l'a pas ; busybox non plus). Le canal
  d'erreur (D-D) **diagnostique déjà** ce cas — à confirmer sur un vrai serveur Linux.

## 4. Points de durcissement à relire (issus de cette session)

- **🔬 SPEC-01 — perf `NSHostingView`-par-ligne** sur très grande liste visible :
  réutilisation des vues + reload sur changement de structure en place ; **mesurer** sur
  un dossier à ~100 k lignes visibles (rare, la liste est O(visible)). Vérifier aussi la
  cadence de `reloadData` pendant un scan (tri qui change à 10 Hz).
- **🔬 SPEC-02/04 — ré-entrance & concurrence de `invalidate`** : un 2ᵉ
  `DirectoryScanner` tourne en place pendant que FSEvents peut re-signaler ; `invalidate`
  n'est pas ré-entrant (gaté `phase != .scanning`, mais deux `refreshDirty` concurrents ?).
  Relire le cas « changement disque pendant un Refresh ».
- **🔬 SPEC-03 — exact + suppression + hardlinks** : `subtreeExtensions` (delete A6) ne
  dédupe pas ; en mode Exact, supprimer un sous-arbre contenant un hardlink dont l'autre
  lien est **dehors** peut sur/sous-compter le panneau File-types. Cas triple-edge, à
  documenter/tester.
- **🔬 SPEC-04 — cycle de vie `FSWatcher`** : `passUnretained` + `stop()` en `deinit` ;
  confirmer qu'aucun callback ne survient après `FSEventStreamInvalidate` (fenêtre
  in-flight sur la dispatch queue). Envisager `passRetained` + release callback si doute.
- **🔬 SPEC-03 — Réconciliation** : `purgeable` = estimation Apple ; **snapshots
  comptés, pas mesurés** ; corbeille via marche `~/.Trash` + `.Trashes/<uid>` (droits ?).
  `delta` peut être négatif si le scan dépasse « used » (attribution) → déjà signalé par
  `scanExceedsUsed`.
